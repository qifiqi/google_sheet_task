# 定时任务蓝屏问题修复

## 问题原因
1. **多实例重复执行**: 两个调度器实例同时在 00:00 执行清理任务
2. **资源消耗过大**: 批量删除 1000 条/批次，间隔仅 0.1 秒，导致数据库/CPU/内存压力过大
3. **阻塞主进程**: 清理任务在 Flask 主进程中执行，影响正常业务

## 修复方案

### 1. 添加分布式锁（防止重复执行）
- 在 `scheduled_tasks` 表添加 `is_running` 和 `running_instance_id` 字段
- 使用乐观锁机制，确保同一时刻只有一个实例执行任务

### 2. 优化清理策略（降低系统负载）
- 批次大小: 1000 → 200 条
- 批次间延迟: 0.1 秒 → 2 秒
- 支持自定义参数配置

### 3. 独立进程执行（避免阻塞主进程）
- 定时任务在独立子进程中执行
- 不影响 Flask 主进程处理用户请求
- 自动释放锁和资源

## 部署步骤

### 步骤 1: 执行数据库迁移
```bash
# 连接到生产数据库
psql -h 172.18.20.17 -U postgres -d googlesheet_validator

# 执行迁移脚本
\i migrations/add_scheduled_task_lock_fields.sql
```

### 步骤 2: 重启应用
```bash
# 停止所有运行的实例
pkill -f "python run.py"
pkill -f "gunicorn"

# 启动应用
python run.py
# 或使用 gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 'app:create_app()'
```

### 步骤 3: 验证
检查日志确认只有一个实例执行清理任务：
```bash
tail -f logs/app.log | grep "定时任务"
```

## 注意事项
- 确保只启动一个应用实例，避免多实例问题
- 如需多实例部署，分布式锁会自动协调执行
- 清理任务执行时间会变长（约 5-10 倍），但不会影响系统稳定性
