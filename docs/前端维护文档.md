## 认证与登录状态

### Token 存储

| Key | 说明 |
|-----|------|
| `access_token` | JWT 访问令牌，有效期 2 小时 |
| `refresh_token` | 刷新令牌，有效期 7 天 |

两者均存储在 `localStorage`。

### useAuth.js

所有认证操作统一通过 `useAuth()` composable：

```js
const { user, permissions, isLoggedIn, login, logout, fetchUser, hasPermission } = useAuth()
```

- `user` / `permissions` 是模块级 `ref`，整个应用共享同一份状态
- 页面刷新后内存状态清空，`AppLayout.vue` 的 `onMounted` 会自动调用 `fetchUser()` 恢复
- `hasPermission(code)` 用于判断当前用户是否拥有某权限

### axios 拦截器（api/index.js）

- 请求拦截：自动从 localStorage 读取 `access_token` 并注入 `Authorization: Bearer <token>` 头
- 响应拦截：遇到 401 时自动用 `refresh_token` 换新 token 并重试原请求；refresh 失败则清除 token 跳转登录页

## 路由与权限守卫

路由定义在 `router/index.js`，每条路由通过 `meta.permission` 绑定所需权限码：

```js
{ path: 'admin/config', name: 'AdminConfig', component: ..., meta: { permission: 'config:view' } }
```

守卫逻辑：
1. 无 token → 跳 `/login`
2. 有 token 但 `user.value` 为空 → 调用 `fetchUser()` 恢复用户信息
3. 路由有 `meta.permission` 且用户不具备该权限 → 跳 `/403`

无 `meta.permission` 的路由，登录后即可访问。

## 导航菜单

侧边栏（`AppSidebar.vue`）从后端 `/api/meta/nav` 动态加载，后端已按当前用户权限过滤。

菜单结构存储在数据库 `system_configs` 表，key 为 `nav_menu`，管理员可直接编辑 JSON 修改菜单。

## 权限指令

```html
<!-- 无权限时该元素会被从 DOM 中移除 -->
<el-button v-permission="'user:manage'">新增用户</el-button>
```

指令定义在 `directives/permission.js`，在 `main.js` 全局注册。

## 常见维护操作

### 新增页面并绑定权限

1. 在 `views/` 下创建 Vue 文件
2. 在 `router/index.js` 添加路由，设置 `meta.permission`
3. 在后端 `system_configs.nav_menu` 的 JSON 里添加菜单项（带 `permission` 字段）
4. 确认 `app/config.py` 的 `PERMISSIONS` 里有对应权限码，没有则补充并重启后端

### 新增 API 调用

在对应的 `api/*.js` 文件里添加函数，统一使用 `api`（`api/index.js` 导出的实例），不要直接用 `axios`。

### 修改导航菜单

登录管理员账号 → 系统配置 → 找到 `nav_menu` 配置项 → 编辑 JSON → 保存。刷新页面生效。

### 本地开发

```bash
cd frontend
npm install
npm run dev   # 启动开发服务器，默认 http://localhost:3000
```

Vite 配置了代理，`/api`、`/backtest-training` 等请求会转发到后端 `http://localhost:5000`。
