### 6.1 基础序列口径

后端先把传入的日收益序列转成净值序列，再基于净值计算年收益、月收益、回撤和各类比率。

1. 净值 `net_value`

当前实现：

```text
net_value = 1 * (1 + return)
```

说明：

- 指数序列使用 `index_return`。
- 策略序列使用 `start_return`。
- 代码位置：`XPLAnalyzer._calculate_metrics_v1()`、`XPLAnalyzer.get_xpl()`

2. 月收益率 `monthly_return`

当前实现：

```text
monthly_return = 月末净值 / 比较基准净值 - 1
```

比较基准：

- 若是首月，取当月第一个数据点。
- 否则取上个月最后一个数据点。

代码位置：

- `XPLAnalyzer.calculate_monthly_return_data()`

3. 年收益率 `annual_return`

当前实现：

```text
annual_return = 本年末净值 / 基准净值 - 1
```

比较基准：

- 若是首年，取当年第一个数据点。
- 否则取上一年最后一个数据点。

代码位置：

- `XPLAnalyzer.calculate_year_returns()`

4. 年化收益率 `annualized_return`

当前实现：

```text
annualized_return = (期末净值 / 期初净值) ^ (365 / 持有天数) - 1
```

代码位置：

- `XPLAnalyzer.annualized_rate_return()`

5. 最大回撤 `drawdown`

当前实现：

```text
drawdown = (历史最高净值 - 当前净值) / 历史最高净值
```

说明：

- 逐点回看截至当前时点的历史最高净值。
- 分别计算年度最大回撤和全区间最大回撤。

代码位置：

- `XPLAnalyzer.calculate_max_drawdown_by_year_and_total()`

### 6.4 回撤类字段

1. `max_drawdown`

含义：

- 年最大超额回撤

当前导出实现：

```text
先筛出 annualized_return_diff > 0 的年份
对这些年份分别计算：
    start_drawdown - index_drawdown
取其中最大值作为 max_drawdown
导出展示时再格式化为负值
```

说明：

- 这里不是直接对“超额净值曲线”算回撤，而是对“跑赢年份的年度回撤差”取最大值。
- 导出时使用 `f"-{max_drawdown:.2%}"`，所以展示值为负号格式。

来源：

- `XPLAnalyzer.format_export_file_data()`

2. `excess_drawdown_winning_rate`

含义：

- 超额回撤胜率

公式：

```text
excess_drawdown_winning_rate =
策略年度回撤 < 指数年度回撤 的年份数 / 总年份数
```

说明：

- 这里比较的是年度最大回撤，回撤越小越优。

来源：

- `XPLAnalyzer.calculate_excess_drawdown_winning_rate()`

3. `start_drawdown`

含义：

- 策略全区间最大回撤

公式：

```text
drawdown = (历史最高净值 - 当前净值) / 历史最高净值
```

来源：

- `XPLAnalyzer.calculate_max_drawdown_by_year_and_total()`
- 导出读取 `start_maximum_drawdown['total_maximum_drawdown']['drawdown']`

4. `start_maximum_number_of_backtest_repair_days`
5. `excess_maximum_number_of_backtest_repair_days`

含义：

- 最大修复天数
- 超额最大修复天数

当前实现：

```text
previous_max = expanding_max(net_value).shift(1)
maximum_number_of_backtest_repair_days = previous_max 众数的出现次数最大值
```

超额口径：

```text
excess_net_value = start_net_value - index_net_value
excess_maximum_number_of_backtest_repair_days =
    对 excess_net_value 序列按同样逻辑计算
```

说明：

- 代码实现与“传统从回撤开始到创新高结束的修复天数”定义并不完全一致。
- 当前是按 `previous_max` 出现频次最大值计算，应以此为准。

来源：

- `XPLAnalyzer.maximum_number_of_backtest_repair_days()`
- `XPLAnalyzer._calculate_metrics_v1()`

### 6.5 比率类字段

1. `index_sharpe_ratio`
2. `start_sharpe_ratio`
3. `index_xpl`
4. `start_xpl`

含义：

- 夏普比率 / XPL 汇总值

公式：

```text
sharpe_ratio = avg_monthly_return * 12 / (monthly_std_dev * sqrt(12))
```

等价写法：

```text
sharpe_ratio = 年化平均收益 / 年化标准差
```

说明：

- 无风险利率按 0 处理。
- `get_xpl()` 返回的就是这一组汇总指标，文档中的 `index_xpl` / `start_xpl` 可视为该口径输出。

来源：

- `XPLAnalyzer.calculate_sharpe_for_period()`
- `XPLAnalyzer.get_xpl()`

5. `index_kama_ratio`
6. `start_kama_ratio`

含义：

- 卡玛比率

公式：

```text
kama_ratio = annualized_return / drawdown
```

说明：

- `all` 汇总口径中，代码把 `drawdown` 取为各年度最大回撤中的最大值。

来源：

- `XPLAnalyzer.calculate_kama_ratio()`

7. `index_sotino_ratio`
8. `start_sotino_ratio`

含义：

- 所提诺比率

公式：

```text
sotino_ratio =
mean(monthly_return) * 12
/ (std(月收益率中小于 0 的部分) * sqrt(12))
```

来源：

- `XPLAnalyzer.calculate_sotino_ratio()`

### 6.6 与 C5 导出字段的对应关系补充

以下字段虽然在 C5 导出表里表现为结果列，但其值来自后端计算或后端二次整理，而不是单纯“原样抄表”：

| 字段 | 说明 |
| --- | --- |
| `start_avg_monthly_return` | 来自 `start_sharpe_ratios['all']['avg_monthly_return']` |
| `start_monthly_std_dev` | 来自 `start_sharpe_ratios['all']['monthly_std_dev']` |
| `start_annual_std_dev` | 来自 `start_sharpe_ratios['all']['annual_std_dev']` |
| `start_xpl` | 来自 `get_xpl()` 返回的汇总结果 |
| `index_avg_monthly_return` | 来自 `index_sharpe_ratios['all']['avg_monthly_return']` |
| `index_monthly_std_dev` | 来自 `index_sharpe_ratios['all']['monthly_std_dev']` |
| `index_annual_std_dev` | 来自 `index_sharpe_ratios['all']['annual_std_dev']` |
| `index_xpl` | 来自 `get_xpl()` 返回的汇总结果 |
| `return_beats` / `dd_beats` 等扩展分析字段 | 建议统一按本节口径理解，优先以后端分析结果为准 |
