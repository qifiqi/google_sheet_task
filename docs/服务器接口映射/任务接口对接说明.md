# 任务接口对接说明

## 1. 文档目的

本文档用于提供给外部服务做接口联调。

本文只包含以下接口：

- C3 创建任务接口
- C5 创建任务接口

不包含内容：

- 批量创建任务接口
- 任务删除接口
- 任务列表接口
- 其他内部管理接口

---

## 2. 接口基础信息

### 2.1 服务地址

请将以下地址替换为实际环境地址：

```text
http://{host}:{port}
```

示例：

```text
http://127.0.0.1:5000
```

### 2.2 鉴权方式

当前接口为登录态接口。

调用时需要带上认证信息，通常为以下两种方式之一：

#### 方式一：Cookie 会话

```http
Cookie: session=xxxxxx
```

#### 方式二：前端统一请求头

如果你们网关或前端代理层有额外鉴权头，请按实际部署要求传递。

### 2.3 请求头

标准请求头如下：

```http
Content-Type: application/json
Accept: application/json
```

---

## 3. 通用响应说明

### 3.1 成功响应

```json
{
  "status": "success",
  "task_id": "f3f6e6ae-0c74-4fa3-8b53-8a52dbf0baf8",
  "message": "任务创建并启动成功"
}
```

### 3.2 失败响应

```json
{
  "status": "error",
  "message": "错误原因说明"
}
```

### 3.3 通用返回字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | string | 响应状态，`success` 或 `error` |
| `task_id` | string | 创建成功后的任务 ID |
| `message` | string | 结果说明或错误说明 |

---

## 4. 通用创建任务接口

### 4.1 接口地址

```http
POST /tasks
```

### 4.2 接口说明

该接口用于创建并立即启动一个任务。

不同任务类型通过 `task_type` 区分：

- `google_sheet`：C3 任务
- `google_sheet_C5`：C5 任务

### 4.3 通用请求体结构

```json
{
  "name": "任务名称",
  "description": "任务描述",
  "task_type": "任务类型",
  "config": {}
}
```

### 4.4 顶层字段说明

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 任务名称 |
| `description` | string | 否 | 任务描述 |
| `task_type` | string | 是 | 任务类型 |
| `config` | object | 是 | 任务配置对象 |

---

## 5. C3 创建任务接口

## 5.1 接口定义

### 请求地址

```http
POST /tasks
```

### 任务类型

```json
"task_type": "google_sheet"
```

### 业务说明

C3 用于单个 Google Sheet 模板任务创建。

创建成功后，系统会立即启动任务。

---

## 5.2 C3 请求参数

### 请求体示例

```json
{
  "name": "AAPL",
  "description": "C3任务示例",
  "task_type": "google_sheet",
  "config": {
    "spreadsheet_id": "1AbCdEfGhIjKlMnOpQrStUvWxYz1234567890",
    "title": "C3策略模板",
    "sheet_name": "Sheet1",
    "token_type": "file",
    "token_id": "__random__",
    "token_file": "",
    "token_json": "",
    "proxy_url": null,
    "parameters": [
      [1, 2, 3],
      [10, 20],
      [30, 40]
    ]
  }
}
```

### cURL 示例

```bash
curl -X POST "http://127.0.0.1:5000/tasks" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -b "session=你的登录态Cookie" \
  -d '{
    "name": "AAPL",
    "description": "C3任务示例",
    "task_type": "google_sheet",
    "config": {
      "spreadsheet_id": "1AbCdEfGhIjKlMnOpQrStUvWxYz1234567890",
      "title": "C3策略模板",
      "sheet_name": "Sheet1",
      "token_type": "file",
      "token_id": "__random__",
      "token_file": "",
      "token_json": "",
      "proxy_url": null,
      "parameters": [
        [1, 2, 3],
        [10, 20],
        [30, 40]
      ]
    }
  }'
```

### 字段说明

#### 顶层字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 任务名称。当前系统中建议直接传股票代码，例如 `AAPL`、`600519` |
| `description` | string | 否 | 任务描述 |
| `task_type` | string | 是 | 固定传 `google_sheet` |
| `config` | object | 是 | C3 配置对象 |

#### `config` 字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `spreadsheet_id` | string | 是 | Google Sheet 的唯一 ID |
| `title` | string | 否 | Google Sheet 标题，主要用于展示与记录 |
| `sheet_name` | string | 是 | 工作表名称 |
| `token_type` | string | 是 | Token 类型，支持 `file` 或 `json` |
| `token_id` | string/int | 条件必填 | 当 `token_type=file` 时传入 |
| `token_file` | string | 否 | Token 文件路径。通常可传空字符串 |
| `token_json` | string | 条件必填 | 当 `token_type=json` 时传入完整 JSON 字符串 |
| `proxy_url` | string/null | 否 | 代理地址，没有可传 `null` |
| `parameters` | array | 是 | 参数组合数组，二维数组结构 |

#### `parameters` 字段说明

`parameters` 为二维数组，每一项代表一个参数维度。

示例：

```json
"parameters": [
  [1, 2, 3],
  [10, 20],
  [30, 40]
]
```

含义：

- 第一组参数候选值：`1`、`2`、`3`
- 第二组参数候选值：`10`、`20`
- 第三组参数候选值：`30`、`40`

系统会自动做笛卡尔积组合执行。

## 6. C5 创建任务接口

## 6.1 接口定义

### 请求地址

```http
POST /tasks
```

### 任务类型

```json
"task_type": "google_sheet_C5"
```

### 业务说明

C5 用于带股票列表、参数 2、参数 3、多 Sheet 配置的任务创建。

创建成功后，系统会立即启动任务。

---

```json
{

    "sheets": [
      {
        "spreadsheet_id": "1XxYyZzAaBbCcDdEeFf112233445566778899",
```


### 字段说明

#### 顶层字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 任务名称 |
| `description` | string | 否 | 任务描述 |
| `task_type` | string | 是 | 固定传 `google_sheet_C5` |
| `config` | object | 是 | C5 配置对象 |

#### `config` 字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `token_type` | string | 是 | Token 类型，支持 `file` 或 `json` |
| `token_id` | string/int | 条件必填 | `token_type=file` 时传入 |
| `token_file` | string | 否 | Token 文件路径，通常可传空字符串 |
| `token_json` | string | 条件必填 | `token_type=json` 时传入完整 JSON 字符串 |
| `proxy_url` | string/null | 否 | 代理地址 |
| `count_mode` | string | 是 | 计数模式，支持 `total` 或 `n_plus_1` |
| `market_type` | string | 是 | 市场类型，`cn` 表示 A 股，`us` 表示美股 |
| `price_mode` | string | 是 | 价格模式，`kp_price` 表示开盘价，`sp_price` 表示收盘价 |
| `date_range_mode` | array | 否 | 日期范围模式，可传 `full`、`recent`，也可同时传两个 |
| `start_date` | string | 是 | 开始日期，格式 `YYYY-MM-DD` |
| `end_date` | string | 是 | 结束日期，格式 `YYYY-MM-DD` |
| `exclude_years` | array | 否 | 最近 N 年模式下需要排除的年份序号列表 |
| `parameters` | array | 是 | 三组参数数组 |
| `sheets` | array | 是 | Google Sheet 目标表列表 |

#### `parameters` 字段说明

`parameters` 在 C5 中为三维结构：

```json
"parameters": [
  ["600519", "000001"],
  [5, 10],
  [20, 30]
]
```

含义如下：

| 下标 | 说明 |
| --- | --- |
| `parameters[0]` | 股票代码列表 |
| `parameters[1]` | 参数 2 列表 |
| `parameters[2]` | 参数 3 列表 |

#### `sheets` 字段说明

```json
"sheets": [
  {
    "spreadsheet_id": "1XxYyZzAaBbCcDdEeFf112233445566778899",
    "sheet_name": "Sheet1",
    "title": "C5主表"
  }
]
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `spreadsheet_id` | string | 是 | Google Sheet ID |
| `sheet_name` | string | 是 | 工作表名称 |
| `title` | string | 否 | 表标题，便于结果识别 |

### 兼容说明

当前服务端同时兼容以下两种排除年份字段：

- `exclude_years`
- `exclude_recent_years`

建议联调时统一使用：

```json
"exclude_years": [1, 2]
```

## 7. 联调建议

### 7.1 最小联调闭环

建议外部服务按以下顺序联调：

1. 登录并拿到会话
2. 调用 `POST /tasks` 创建 C3 任务
3. 记录返回的 `task_id`
4. 再调用 `POST /tasks` 创建 C5 任务
5. 记录返回的 `task_id`

### 7.2 推荐校验点

创建后建议至少校验以下内容：

- HTTP 状态码是否为 `200`
- 响应体 `status` 是否为 `success`
- 是否返回 `task_id`
- `task_id` 是否为合法 UUID 字符串

## 9. C5 导出数据接口建议

## 9.1 背景说明

当前系统里：

- C3 已有类似“结果导出/回传”的外部对接思路
- C5 目前没有现成的第三方结果回传接口

如果需要让外部服务接收 C5 任务产出，建议对方服务端提供一个结果接收接口，由我方在 C5 任务执行成功后主动调用。

也就是说，这一节不是当前系统已经存在的接口，而是建议对方服务端新增的接口规范。

---

## 9.2 建议接口定义

### 接口名称

C5 任务结果导出接口

### 请求地址

```http
POST /api/task/c5/export
```

### 完整示例地址

```text
http://{server-host}/api/task/c5/export
```

例如：

```text
http://example.com/api/task/c5/export
```

### 请求头

```http
Content-Type: application/json
Accept: application/json
```

如果对方服务需要签名或 token，也可以额外加：

```http
Authorization: Bearer xxxxxx
```

---

## 9.3 接口用途

该接口用于接收我方系统导出的 C5 任务计算结果。

建议在以下时机调用：

- C5 任务全部执行完成后，按任务维度整体回传
- 或者每个 step 执行成功后，按单步结果逐条回传

建议优先采用：

- 单步明细回传

原因：

- 失败重试更容易
- 对账更清晰
- 外部服务更容易增量落库

---

## 9.4 推荐请求体结构

### 方案一：按单步明细回传

这是更推荐的方案。

```json
{
  "source_system": "google-parameter-validator",
  "task_type": "google_sheet_C5",
  "task_id": "8a17c8a2-c4d3-4f4e-bb8f-0f4f8c6f5d2a",
  "task_name": "C5任务示例",
  "step_index": 0,
  "success": true,
  "exported_at": "2026-04-21T10:30:00+08:00",
  "parameters": {
    "stock_code": "600519",
    "A1": 5,
    "B1": 20,
    "year": "2025-2023",
    "Kline_key": "2025-2023",
    "kline_start_date": "2023-01-03",
    "kline_end_date": "2025-12-31"
  },
  "result": {
    "sheet_id_1__C5主表": {
      "D2": 1.23,
      "E2": 4.56,
      "F2": 7.89,
      "index_return_xpl": 0.11,
      "start_return_xpl": 0.22
    }
  }
}
```

### 方案二：按整个任务批量回传

如果对方更适合一次接收整个任务的数据，也可以采用：

```json
{
  "source_system": "google-parameter-validator",
  "task_type": "google_sheet_C5",
  "task_id": "8a17c8a2-c4d3-4f4e-bb8f-0f4f8c6f5d2a",
  "task_name": "C5任务示例",
  "status": "completed",
  "exported_at": "2026-04-21T10:30:00+08:00",
  "items": [
    {
      "step_index": 0,
      "success": true,
      "parameters": {
        "stock_code": "600519",
        "A1": 5,
        "B1": 20,
        "year": "2025-2023",
        "Kline_key": "2025-2023",
        "kline_start_date": "2023-01-03",
        "kline_end_date": "2025-12-31"
      },
      "result": {
        "sheet_id_1__C5主表": {
          "D2": 1.23,
          "E2": 4.56,
          "index_return_xpl": 0.11,
          "start_return_xpl": 0.22
        }
      }
    }
  ]
}
```

## 9.5 字段说明

### 顶层字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `source_system` | string | 是 | 数据来源系统标识，建议固定值 |
| `task_type` | string | 是 | 固定传 `google_sheet_C5` |
| `task_id` | string | 是 | 我方系统任务 ID |
| `task_name` | string | 是 | 任务名称 |
| `step_index` | int | 条件必填 | 单步回传时必填，表示第几步 |
| `status` | string | 条件必填 | 整体任务回传时可传 `completed` |
| `success` | bool | 条件必填 | 单步回传时表示本步是否成功 |
| `exported_at` | string | 是 | 导出时间，建议 ISO8601 格式 |
| `parameters` | object | 条件必填 | 单步回传时的输入参数对象 |
| `result` | object | 条件必填 | 单步回传时的结果对象 |
| `items` | array | 条件必填 | 整体任务回传时的结果明细数组 |

### `parameters` 字段说明

| 字段 | 类型 | 必填 | 中文说明 |
| --- | --- | --- | --- |
| `stock_code` | string | 是 | 股票代码 |
| `A1` | number/string | 是 | C5 参数 2，对应内部组合参数 |
| `B1` | number/string | 是 | C5 参数 3，对应内部组合参数 |
| `year` | string/number | 是 | 当前计算窗口标识，可能是年份，也可能是区间标识 |
| `Kline_key` | string/number | 是 | K 线缓存键 |
| `kline_start_date` | string | 否 | 本次使用 K 线开始日期 |
| `kline_end_date` | string | 否 | 本次使用 K 线结束日期 |

### `result` 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `sheet_id_1__C5主表` | object | 某个目标 Sheet 的结果对象，键名格式建议保留 |
| `D2` / `E2` / `F2` | number/string | 来自 Google Sheet 的结果单元格值 |
| `index_return_xpl` | number | 指数收益率衍生结果 |
| `start_return_xpl` | number | 起始收益率衍生结果 |

说明：

- `result` 内部的单元格字段不一定固定，可能会随着模板变化而变化
- 因此建议对方服务端采用“动态 JSON 字段存储”方式，不要写死字段结构

## 9.6 对方服务端建议返回

### 成功响应示例

```json
  "code": 0,
  "message": "ok",
  "data": {

  }
```

### 失败响应示例

```json
{
  "code": 4001,
  "message": "参数缺失：task_id"
}
```

## 9.7 建议对方服务端最少接收哪些字段

如果对方服务暂时不想一次性支持完整结构，至少建议先支持以下核心字段：

```json
{
  "task_type": "google_sheet_C5",
  "task_id": "8a17c8a2-c4d3-4f4e-bb8f-0f4f8c6f5d2a",
  "task_name": "C5任务示例",
  "step_index": 0,
  "success": true,
  "parameters": {
    "stock_code": "600519",
    "A1": 5,
    "B1": 20,
    "year": "2025-2023"
  },
  "result": {
    "sheet_id_1__C5主表": {
      "index_return_xpl": 0.11,
      "start_return_xpl": 0.22
    }
  }
}
```

也就是说，对方服务端最少需要能接收：

- 任务标识
- 步骤标识
- 输入参数
- 输出结果

## 9.8 给对方服务端的接口摘要

如果你要直接发给对方服务端一句话说明，可以用下面这段：

```text
请你们提供一个 POST 接口，例如 /api/task/c5/export，
用于接收我方 C5 任务的导出结果。
接口请求体建议至少包含 task_id、task_name、step_index、success、
parameters（包含 stock_code、A1、B1、year）、
result（包含每个 sheet 的结果对象）。
返回 code=0 表示接收成功。
```
