## API 层设计

两个 axios 实例：
- `api` — baseURL `/api`，用于大部分接口
- `rawApi` — 无 baseURL，用于 `/admin/api/*`、`/backtest-training/api/*`、`/backtest-multi-product/api/*`

共享拦截器：
- 请求：自动附加 `Authorization: Bearer <token>`
- 响应：自动解包 `res.data`；401 时自动刷新 token 并重试

## 主题系统

- CSS Custom Properties 定义在 `:root` 和 `[data-theme='dark']`
- `useTheme` composable 切换 `document.documentElement.dataset.theme`
- 持久化到 `localStorage.theme`
- Element Plus 变量通过 `--el-*` 覆盖实现统一

---

## 响应式策略

三级断点：
- Mobile: < 768px
- Tablet: 768px ~ 1199px
- Desktop: >= 1200px

实现方式：
1. **SCSS mixin** — `@include mobile { }` / `@include tablet { }` / `@include desktop { }`
2. **JS composable** — `useResponsive()` 返回 `isMobile` / `isTablet` / `isDesktop`
3. **组件内置** — DataTableCard 自动简化分页，FilterToolbar 自动堆叠

## 性能优化措施

| 措施 | 实现位置 |
|------|---------|
| 路由懒加载 | `router/index.js` — 所有路由使用 `() => import()` |
| 轮询防并发 | `usePolling` — inFlight 锁 + 可见性感知 |
| 输入防抖 | `FilterToolbar` — input 类型 300ms 防抖 |
| 虚拟滚动 | `LogViewer` — 超过 500 条自动启用 |
| 大 JSON 折叠 | `CodeBlock` — 超过 10KB 折叠显示 |
| Chart.js CDN 懒加载 | `useChartJs` — 首次使用时动态加载 |
| Element Plus 按需导入 | `vite.config.js` — unplugin-vue-components |

## 开发命令

```bash
# 安装依赖
cd frontend && npm install

# 开发服务器 (port 3000, 代理 API 到 Flask 5000)
npm run dev

# 生产构建
npm run build

# 构建产物在 frontend/dist/
```

## 新增页面检查清单

1. 在 `views/` 下创建 `.vue` 文件
2. 在 `router/index.js` 注册路由（懒加载 + meta.permission）
3. 如需新 API，在 `api/` 下添加模块
4. 使用已有组件：PageToolbar, FilterToolbar, DataTableCard, StatCardGrid 等
5. 使用 `usePolling` 替代 `setInterval`
6. 使用 `useResponsive` 处理移动端适配
7. `npm run build` 验证无编译错误
