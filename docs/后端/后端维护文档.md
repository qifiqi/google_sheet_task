# 后端维护文档

## 技术栈

- Python 3.10+
- Flask 2.x
- SQLAlchemy + Flask-Migrate
- SQLite（默认）/ PostgreSQL / MySQL
- JWT（PyJWT）
- APScheduler（定时任务）
- gspread（Google Sheet 客户端）

---

## 启动方式

```bash
# 安装依赖
pip install -r requirements.txt

# 启动（开发/生产均用此命令）
python run.py
```

启动时 `run.py` 会自动执行：
1. `db.create_all()` — 建表
2. `ensure_google_sheet_token_schema()` — 补充旧库缺失字段
3. `ensure_user_token_version_schema()` — 补充 `user.token_version` 字段（用于单点登录）
4. `reset_google_sheet_token_occupancy()` — 清零 token 占用计数
5. `reset_google_sheet_occupancy()` — 清理 Google Sheet 占用状态
6. `init_config2()` — 写入默认系统配置
7. `init_rbac()` — 幂等初始化权限、角色、admin 用户、导航菜单
8. `check_and_cleanup_dead_tasks()` — 重置挂死任务
9. `init_scheduler()` — 启动定时任务调度器
10. `init_task_watchdog()` — 启动任务看门狗线程

## 认证系统

### 开关

通过环境变量控制：

```bash
AUTH_ENABLED=true   # 启用 JWT 认证
AUTH_ENABLED=false  # 关闭认证（开发调试用，所有接口放行）
```

关闭时，`login_required` 装饰器会注入一个拥有全部权限的 mock 用户到 `g.current_user`，下游代码无需判断。

### 装饰器用法

```python
@app.route('/api/xxx')
@login_required                          # 验证 JWT，设置 g.current_user
@permission_required('task:view')        # 检查权限（OR 逻辑，传多个任一满足即可）
def my_view():
    user = g.current_user
```

### Token 有效期

| 类型 | 有效期 |
|------|--------|
| access_token | 2 小时 |
| refresh_token | 7 天 |

JWT Secret 从 `system_configs` 表读取，key 为 `JWT_SECRET_KEY`，默认值 `change-me-in-production`（生产环境务必修改）。

### 单点登录机制（token_version）

项目当前采用“同账号单会话”策略：

1. 用户每次 `POST /api/auth/login` 成功后，后端都会把 `user.token_version + 1`
2. 新签发的 access/refresh token 都会写入最新 `token_version`
3. `@login_required` 会比对 access_token 中的 `token_version` 与数据库值
4. `/api/auth/refresh` 同样会比对 refresh_token 的 `token_version`
5. 版本不一致即返回 401，前端应清理 token 并回到登录页

这意味着：用户在新端登录后，旧端会在下一次请求时自动失效（符合单点登录预期）。

### 鉴权装饰器最佳实践

```python
@bp.route('/api/secure-resource', methods=['POST'])
@login_required
@permission_required('task:create', 'task:manage')
def secure_resource():
    ...
```

- `@login_required` 放在前面，确保先拿到 `g.current_user`
- `@permission_required` 支持多个权限码，逻辑为 OR（满足任一即可）
- 公开接口不要加 `@permission_required`，只保留必要鉴权

## 导航菜单

菜单结构存储在 `system_configs` 表，key 为 `nav_menu`，值为 JSON 字符串。

每个菜单项可带 `permission` 字段，后端在 `/api/meta/nav` 接口返回时按当前用户权限过滤。

**修改菜单**：直接在数据库或管理后台系统配置页编辑 `nav_menu` 的值，无需重启。

默认菜单结构定义在 `app/config.py` 的 `NAV_MENU`，仅首次启动时写入，后续不覆盖。

## 任务系统

### 执行链路

```
前端提交参数
  → POST /api/tasks
  → task_manager.create_task()
  → task_manager.start_task()
  → 后台线程 _execute_google_sheet_task() / _execute_google_sheet_C4_task() 等
  → google_sheet_service.execute_task()
  → google_sheet_client（网络 IO）
  → 写 TaskResult / TaskLog
  → 更新 Task.status
```

### C31 批量任务

C31 不是独立执行器，前端提交后由 `task_manager.batch_create_and_start_task()` 拆分为多个 C3 子任务并行执行。

### 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待执行 |
| `running` | 执行中 |
| `completed` | 完成 |
| `failed` | 失败 |
| `cancelled` | 已取消 |

### 看门狗

`task_watchdog.py` 定期检查：
- `running` 任务长时间无日志 → 标记失败
- `error_message` 含 `[NETWORK_RETRYABLE]` 前缀的失败任务 → 自动重启

## 常见维护操作

### 新增权限码

1. 在 `app/config.py` 的 `PERMISSIONS` 列表追加 `(group, code, name, route_path)`
2. 重启应用，`init_rbac()` 自动入库
3. 在管理后台给对应角色分配新权限

### 新增 API 接口并保护

```python
@bp.route('/api/xxx', methods=['GET'])
@login_required
@permission_required('task:view')
def my_api():
    ...
```

### 数据库字段变更

项目不完全依赖 Flask-Migrate，优先在 `run.py` 里参考 `ensure_google_sheet_token_schema()` 的写法补字段，或在 `scripts/` 目录创建迁移脚本手动执行。

### 修改 JWT Secret

在管理后台系统配置页修改 `JWT_SECRET_KEY` 的值，或直接更新数据库 `system_configs` 表。修改后所有已签发的 token 立即失效，用户需重新登录。

### 鉴权开关操作

```bash
# 推荐：生产环境开启
AUTH_ENABLED=true

# 仅本地调试可关闭（会绕过鉴权）
AUTH_ENABLED=false
```

切换后请重启 `python run.py` 使配置生效。

### 查看运行日志

```bash
tail -f logs/app.log
```

日志级别通过环境变量 `LOG_LEVEL` 控制，默认 `INFO`。
