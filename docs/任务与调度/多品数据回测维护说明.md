# 多品数据回测维护说明

本文记录“多品数据回测”的实现约定，后续修改时优先按这里的链路检查，避免只改页面或只改 service 导致任务恢复、结果预览、比例计算不一致。

## 功能定位

- 页面入口：`/backtest-multi-product`
- 任务类型：`backtest_multi_product`
- 目标：在一个任务中执行多个产品，每个产品使用独立 Google Sheet，按同一 K 线日期区间执行回测。
- 参数对齐：按行号对齐。产品 1 第 N 行参数、产品 2 第 N 行参数共同组成“参数方案 N”。
- 比例规则：每个产品有 `ratio`，必须是非负数字；合计不强制等于 `100%`。
- 全局预览：按参数方案切换，列结构为：
  - 每个产品合并表头下展示 `指数`、`模型结果`、`模型结果（比例）`
  - ...
  - `比例计算-指数`
  - `比例计算-结果`

## 主要文件

业务实现尽量放在独立文件中：

- `app/routes/backtest_multi_product.py`
  - 页面路由
  - 结果分页接口
  - 全局预览接口
  - 比例保存接口
  - Excel 导出接口
- `app/services/backtest_multi_product_service.py`
  - `BacktestMultiProductService`
  - config 校验
  - 比例校验
  - 多产品执行
  - 全局预览 payload 构建
  - 加权计算
- `templates/backtest_multi_product/`
  - `create.html`
  - `list.html`
  - `detail.html`
  - `result.html`
  - `global_preview.html`
- `tests/test_backtest_multi_product.py`
  - 比例校验
  - 参数行对齐校验
  - 全局预览加权计算

必要接线分散在这些现有文件中：

- `app/routes/__init__.py` 注册蓝图
- `app/services/task/runtime.py` 启动 `BacktestMultiProductService`，处理多 Sheet 锁
- `app/services/task/creation.py` 标记回测 token 类型，创建前校验 config
- `app/services/task/restart.py` 重启前检查 Sheet 占用
- `app/services/task/occupancy.py` 从 `products[*].sheet` 收集 Google Sheet registry 占用
- `app/utils/task_authorization.py` 将 `backtest_multi_product` 纳入回测权限作用域
- `app/navigation.py`、`app/config.py`、`app/startup.py`、`app/routes/meta_api.py` 维护导航和页面权限入口

## Config 结构

任务创建时 `Task.config` 的核心结构如下：

```json
{
  "start_date": "2024-01-01",
  "end_date": "2024-12-31",
  "token_type": "file",
  "token_id": 1,
  "token_task_type": "backtest_training",
  "products": [
    {
      "product_index": 0,
      "product_name": "产品1",
      "stock_code": "AAPL",
      "market_type": "en",
      "ratio": "50",
      "sheet": {
        "spreadsheet_id": "...",
        "sheet_name": "data",
        "title": "C3 model"
      },
      "parameters": [
        ["0.0350%", "1", "2", "3", "4", "5", "6", "7"]
      ]
    }
  ]
}
```

约束：

- `products` 至少 2 个。
- 所有产品的 `parameters.length` 必须一致。
- 所有产品的 `ratio` 必须是非负数字，合计不要求为 `100`。
- `start_date <= end_date`。
- 每个产品必须有 `stock_code` 和 `sheet.spreadsheet_id`。
- `market_type` 规范化为 `cn` 或 `en`。
- 多品任务复用回测 Token 池，所以 `token_task_type` 是 `backtest_training`。

## 执行链路

1. 创建页提交 `/api/tasks`，`task_type=backtest_multi_product`。
2. `TaskCreationMixin._normalize_task_config_for_type()` 调用 `normalize_multi_product_config()` 做创建前校验。
3. `TaskRuntimeMixin.start_task()`：
   - 提取所有 `products[*].sheet.spreadsheet_id`
   - 检查是否已有 `backtest_training` 或 `backtest_multi_product` 正在使用同一 Sheet
   - 为所有 Sheet 创建文件锁
   - 创建执行线程 `_execute_backtest_multi_product_task()`
4. `BacktestMultiProductService.execute_task()`：
   - 合并系统 Google Sheet 配置
   - 再次规范化 config
   - 按 `参数方案 -> 产品` 串行执行
5. 每一步保存一条 `TaskResult`，`parameters` 中必须包含：
   - `product_index`
   - `product_name`
   - `stock_code`
   - `ratio`
   - `parameter_group_index`
   - `parameter`
   - `kline`

## 预览和比例计算

全局预览由 `build_multi_product_global_preview_payload(task_id)` 构建。

分组方式：

- 读取所有成功 `TaskResult`
- 按 `parameters.parameter_group_index` 分组
- 每组对应一个“参数方案”

行定义：

- `SUMMARY_ROW_DEFS` 维护展示的指标、来源字段和类型。
- 每行生成所有产品的指数值、模型结果值、单产品按比例折算后的模型结果值。
- 再生成 `weighted_index_value` 和 `weighted_result_value`。

加权规则：

```text
weighted = sum(value * ratio / 100)
```

注意：

- 百分比字符串会按百分比解析，例如 `20% -> 0.2`。
- 普通数字按数值解析。
- 某一产品该指标不可解析或缺失时，加权结果显示 `-`。
- 预览页修改比例后调用：
  - `PUT /backtest-multi-product/api/global-preview/<task_id>/ratios`
- 保存成功会写回 `Task.config.products[*].ratio`，刷新后仍保留。

## 后续修改检查清单

改产品字段时至少检查：

- `templates/backtest_multi_product/create.html` 的产品卡片采集逻辑
- `normalize_multi_product_config()`
- `BacktestMultiProductService._execute_products()`
- `build_multi_product_global_preview_payload()`
- `templates/backtest_multi_product/detail.html` 的参数摘要
- `templates/backtest_multi_product/global_preview.html` 的展示与比例保存

改执行逻辑时至少检查：

- 多 Sheet 锁是否仍完整释放
- `Task.current_step` 和 `Task.total_steps` 是否仍按实际执行步数更新
- 断点重启 `_resolve_resume_start_index()` 是否仍能按 `TaskResult.step_index` 跳过已完成结果
- `TaskResult.parameters` 是否保留 `product_index` 和 `parameter_group_index`

改预览指标时至少检查：

- `SUMMARY_ROW_DEFS`
- `_derive_metrics()`
- `_weighted_display()`
- XLSX 导出 `_build_global_preview_workbook()`
- `tests/test_backtest_multi_product.py`

## 已知注意点

- 多品任务不会修改单品 `backtest_training` 的页面和 service。
- 多品任务和单品任务共享同一回测 Sheet 锁池，任一 Sheet 冲突都会阻止启动。
- 由于每个产品独立 Sheet，创建页必须先识别每个产品的 Sheet，否则无法判断 C3/C5 参数列。
- 当前全局预览按参数方案切换，不做所有方案横向展开，避免宽表不可用。
