# 单产品与多产品全局预览指标空值排障说明

本文用于排查单产品回测和多产品回测全局预览页中的指标空值、横杠和符号异常。

本文以 `app/services/xpl_service.py` 中的 `XPLAnalyzer.format_export_file_data()` 为摘要指标展示基准。该函数保持不变；单产品和多产品全局预览在各自代码中兼容它的字段结构和展示口径。

## 1. 页面和代码入口

### 1.1 单产品全局预览

- 页面：`/backtest-training/global-preview/<task_id>`
- API：`GET /backtest-training/api/global-preview/<task_id>`
- 后端：`app/routes/backtest_training.py`
  - `_build_global_preview_payload()`
  - `_extract_summary_rows()`
  - `_normalize_calculate_metrics_years_for_xpl_export()`
- 页面模板：`templates/backtest_training/global_preview.html`

单产品预览优先把 `TaskResult.result[*].calculate_metrics` 交给
`xpl_analyzer.format_export_file_data()`，再读取其中 20 行摘要指标。

`format_export_file_data()` 对字段完整度要求较高。某个嵌套字段缺失时，单产品预览会进入 `_build_fallback_rows()`，尽量展示仍然可以计算的指标。

### 1.2 多产品全局预览

- 页面：`/backtest-multi-product/global-preview/<task_id>`
- API：`GET /backtest-multi-product/api/global-preview/<task_id>`
- 比例试算 API：`POST /backtest-multi-product/api/global-preview/<task_id>/calculate-ratios`
- 后端：`app/services/backtest_multi_product_service.py`
  - `build_multi_product_global_preview_payload()`
  - `_derive_metrics()`
  - `_derive_year_max_excess_drawdown()`
  - `_build_weighted_product_metrics()`
  - `_build_portfolio_metrics()`
- 页面模板：`templates/backtest_multi_product/global_preview.html`

多产品预览不直接调用 `format_export_file_data()` 生成表格，而是在 `_derive_metrics()` 中读取相同字段并组装为相同的 20 个摘要指标。

多产品每个参数方案包含三种展示层级：

| 层级 | 页面列 | 数据来源 |
| --- | --- | --- |
| 原始单产品 | 每个产品的 `指数`、`模型结果` | 当前产品 `TaskResult.result[*].calculate_metrics` |
| 比例后单产品 | 每个产品的 `模型结果（比例）` | 当前产品收益序列按比例缩放后，再运行 `get_calculate_metrics_v1()` |
| 组合结果 | `比例计算-指数`、`比例计算-结果` | 所有产品按共同日期合成组合收益序列后，再运行 `get_calculate_metrics_v1()` |

## 2. 横杠的含义

页面模板会把空字符串、`null` 或缺少的字段显示为 `-`。

需要先区分两类横杠。

### 2.1 设计上必然出现的横杠

以下指标只有模型结果，没有独立的指数结果。它们的指数列显示 `-` 是正常行为：

- 年化超额收益
- 跑赢年份(百分比)
- 月超额收益胜率
- 平均月超额
- 月超额波动率
- 年最大超额回撤
- 超额回撤胜率
- 年最大回撤
- 最大修复天数
- 超额最大修复天数
- 超额夏普
- 超额所提诺比率

多产品页中的 `比例计算-指数` 也遵循同一规则：只有配置了指数来源字段的指标才会展示指数结果。

### 2.2 需要排查的横杠

如果本应有值的模型列、比例后模型列或组合结果列显示 `-`，说明后端派生结果为空。重点检查：

- `TaskResult.success` 是否为 `true`
- `TaskResult.result` 中是否存在 `calculate_metrics`
- `calculate_metrics` 中对应字段是否存在
- 比例后指标是否可以取得收益序列
- 组合指标是否可以为全部产品构造共同日期收益序列
- 指标计算结果是否为 `NaN` 或 `Infinity`

## 3. 摘要指标来源对照

下表按照 `format_export_file_data()` 的展示顺序列出字段来源。

| 指标 | XPL 字段来源 | 为空或显示横杠的主要原因 |
| --- | --- | --- |
| 年化收益 | `excess_returns[year=all].index_annualized_return`、`start_annualized_return` | 缺少 `year=all` 汇总项，或 `calculate_metrics` 整体为空 |
| 盈利年份百分比 | `index_profit_annual`、`start_profit_annual` | 年度收益序列未生成，或 XPL 计算整体失败 |
| 月盈利百分比 | `index_profit_monthly[year=all].profit_monthly_percentage`、`start_profit_monthly[year=all].profit_monthly_percentage` | 缺少月收益序列，或缺少 `year=all` 汇总项 |
| 平均月收益率 | `index_sharpe_ratios.all.avg_monthly_return`、`start_sharpe_ratios.all.avg_monthly_return` | 月收益序列不足，或夏普汇总项缺失 |
| 月收益率波动率 | `index_monthly_return_volatility`、`start_monthly_return_volatility` | 月收益序列不足，或标准差无法计算 |
| 年化超额收益 | `excess_returns[year=all].annualized_return_diff` | 缺少 `year=all` 汇总项 |
| 跑赢年份(百分比) | `outperform_year` | 年度超额收益序列未生成 |
| 月超额收益胜率 | `monthly_excess_return_percentage[year=all].excess_return` | 月超额收益序列为空，或缺少 `year=all` 汇总项 |
| 平均月超额 | `mean(monthly_excess_returns[*].monthly_excess_return_diff)` | `monthly_excess_returns` 为空 |
| 月超额波动率 | `monthly_excess_volatility` | 月超额收益样本不足，标准差无法计算 |
| 年最大超额回撤 | 见第 4 节 | 缺少年超额收益、缺少年回撤数据，或年份无法对应 |
| 超额回撤胜率 | `excess_drawdown_winning_rate`，展示时取负号 | 年回撤数据未生成；极端情况下分母为空 |
| 年最大回撤 | `start_maximum_drawdown.total_maximum_drawdown.drawdown`，展示时取负号 | 模型回撤结构缺失 |
| 最大修复天数 | `start_maximum_number_of_backtest_repair_days` | 模型净值序列未生成 |
| 超额最大修复天数 | `excess_maximum_number_of_backtest_repair_days` | 指数和模型净值序列未生成 |
| 夏普比率 | `index_sharpe_ratios.all.sharpe_ratio`、`start_sharpe_ratios.all.sharpe_ratio` | 月收益样本不足，或波动率为零 |
| 卡玛比率 | `index_kama_ratio[year=all].kama_ratio`、`start_kama_ratio[year=all].kama_ratio` | 年化收益或最大回撤缺失；最大回撤为零时可能出现非有限值 |
| 所提诺比率 | `index_sotino_ratio[year=all].sotino_ratio`、`start_sotino_ratio[year=all].sotino_ratio` | 下行月收益样本不足，或下行波动率为零 |
| 超额夏普 | `excess_sharp` | 月超额收益样本不足，或月超额标准差为零 |
| 超额所提诺比率 | `excess_of_promissory_note` | 下行月超额收益样本不足，或下行标准差为零 |

## 4. 年最大超额回撤的口径

`format_export_file_data()` 的算法是：

```text
1. 从 excess_returns 中筛出 annualized_return_diff > 0 且 year != all 的年份。
2. 从 index_maximum_drawdown.year_maximum_drawdown 中取这些年份的指数回撤。
3. 从 start_maximum_drawdown.year_maximum_drawdown 中取这些年份的模型回撤。
4. 对同一年计算：模型回撤 - 指数回撤。
5. 取所有年份差值中的最大值。
6. 展示时再加负号。
```

示例：

```text
指数回撤 = 10%
模型回撤 = 6%
内部差值 = 6% - 10% = -4%
展示值 = -(-4%) = 4%
```

### 4.1 字符串年份与数字年份

历史结果中年份可能是 `"2024"`，也可能是 `2024`。如果直接比较，可能导致同一年无法匹配。

当前处理方式：

- 单产品预览在 `_normalize_calculate_metrics_years_for_xpl_export()` 中复制一份指标对象，并把可转换的年份统一为整数，再交给原有 XPL 导出逻辑。
- 多产品预览在 `_derive_year_max_excess_drawdown()` 中通过 `_year_key()` 统一年份键。

`app/services/xpl_service.py` 本身不做改动。

### 4.2 正常零值与异常空值

需要区分：

- 年度收益数据完整，但没有任何 `annualized_return_diff > 0` 的年份：展示 `-0.00%`。这是正常零值。
- 年度收益结构不存在：展示 `-`。这是需要排查的数据缺失。
- 存在正超额年份，但同年指数回撤或模型回撤缺失：展示 `-`。这是年份对应关系或回撤结构缺失。

## 5. 单产品预览为空时怎么查

单产品全局预览链路：

```text
TaskResult
  -> result[*].calculate_metrics
  -> _extract_summary_rows()
  -> format_export_file_data()
  -> 失败时 _build_fallback_rows()
  -> 页面把空字符串显示为 -
```

### 5.1 整列都没有指标

优先检查：

1. 当前 `TaskResult.success` 是否为 `false`。
2. `TaskResult.result` 是否为空。
3. `TaskResult.result` 的第一个业务结果对象是否包含 `calculate_metrics`。
4. `calculate_metrics` 是否为空对象 `{}`。

回测服务会在 `app/services/backtest_training_service.py` 中读取每日指数收益和模型收益，再调用：

```python
calculate_metrics = self.xpl.get_calculate_metrics_v1(_return_date)
```

如果 XPL 指标计算过程抛出异常，`get_calculate_metrics_v1()` 可能返回空对象，单产品预览无法生成摘要行。

### 5.2 只有部分值为空

优先按第 3 节查对应字段。

单产品预览会先走严格的 `format_export_file_data()`。如果任何必需字段缺失，整个格式化过程会抛出异常，然后进入 fallback。fallback 会保留仍然可计算的值，因此“只有部分横杠”通常表示原始 `calculate_metrics` 不完整，而不是页面模板问题。

### 5.3 区间为空

回测区间来自：

```text
excess_returns[year=all].start_end_date
```

如果该字段缺失或不是 `开始时间/结束时间` 格式，严格导出逻辑会失败，单产品预览进入 fallback。此时指标可能仍能展示，但区间为空。

## 6. 多产品预览为空时怎么查

### 6.1 原始单产品的模型结果为空

检查：

1. 当前参数方案下是否存在该产品的成功 `TaskResult`。
2. `TaskResult.parameters.product_index` 是否正确。
3. `TaskResult.result[*].calculate_metrics` 是否存在。
4. 第 3 节中对应字段是否存在。

### 6.2 `模型结果（比例）` 为空

比例后单产品指标优先读取：

```text
TaskResult.result[*].weighted_calculate_metrics
```

当缓存不存在或比例发生变化时，后端会从收益序列重新计算。收益序列读取顺序：

```text
1. TaskResult.return_series_id
2. TaskResultReturn.returns_json
3. 兼容历史结果中的内联 return_date
```

如果收益序列缺失、为空或每行缺少 `date`、`index_return`、`start_return`，比例后指标无法计算，对应列显示 `-`。

### 6.3 `比例计算-指数` 和 `比例计算-结果` 为空

组合指标不是把各产品指标直接相加，而是：

```text
1. 读取每个产品的每日收益序列。
2. 找到所有产品共同拥有的日期。
3. 每个日期先按产品比例合成 index_return 和 start_return。
4. 使用合成后的每日收益序列重新运行 get_calculate_metrics_v1()。
```

因此组合列显示 `-` 的常见原因：

- 当前参数方案缺少任意一个产品的成功结果。
- 任意一个产品缺少收益序列。
- 任意一个产品的收益序列解析后为空。
- 产品之间没有共同日期。
- 合成后的收益序列进入 XPL 计算后返回空对象。

### 6.4 修改比例后出现横杠

检查：

1. 页面是否已经点击“计算预览”。
2. 比例是否为非负数字。
3. 修改比例后，后端是否仍能读取原始收益序列。
4. 比例为 `0` 时，部分比率指标是否因为波动率变为零而无法计算。

比例为 `0` 不会使所有收益类指标为空，但夏普、卡玛、所提诺、超额夏普和超额所提诺等比率类指标可能因为分母为零变成非有限值。

## 7. 非有限值：`NaN` 和 `Infinity`

以下场景可能产生非有限值：

- 月收益样本数量不足
- 月收益标准差为零
- 下行收益标准差为零
- 最大回撤为零
- 月超额收益标准差为零

多产品预览的 `_safe_number()` 会把 `NaN` 和 `Infinity` 转为不可展示值，因此页面显示 `-`。

单产品预览优先走原有 `format_export_file_data()`。该函数使用字符串格式化，某些非有限值可能显示为 `nan` 或 `inf`。如果格式化过程因为其他字段异常进入 fallback，则对应值通常显示为空。排障时应同时把 `nan`、`inf` 和 `-` 视为“指标数学上不可定义”，继续检查样本数量和分母。

## 8. 本轮审计发现并修正的问题

本轮以 `format_export_file_data()` 为基准检查了单产品和多产品全局预览。

已修正：

1. 多产品“年最大超额回撤”原先读取不存在的顶层字段 `calculate_metrics.max_drawdown`，导致页面显示 `-`。现在改为按 XPL 导出口径从年度数据派生。
2. 单产品历史结果中年份为字符串时，可能无法和年度回撤年份匹配。现在单产品预览在调用 XPL 导出逻辑前规范化年份副本。
3. 多产品年度回撤年份可能同时存在字符串和数字。现在通过统一年份键匹配。
4. 多产品“超额回撤胜率”原先直接展示正值；现在按 XPL 导出口径展示负值。
5. 多产品“年最大回撤”原先直接展示正值；现在按 XPL 导出口径展示负值。
6. 多产品在年度数据完整但没有正超额年份时，原先显示 `-`；现在按 XPL 导出口径展示 `-0.00%`。

检查结果：

- 其余摘要指标的数据来源与 `format_export_file_data()` 一致。
- `app/services/xpl_service.py` 未改动。

## 9. 已知边界和后续注意事项

以下行为本轮记录但不修改：

1. `format_export_file_data()` 是严格格式化函数。任何必需字段缺失都可能触发异常。单产品预览通过 fallback 尽量保留可展示指标。
2. `format_export_file_data()` 的回撤明细区使用 `zip(index_year_maximum_drawdown, start_year_maximum_drawdown)` 按列表位置配对。正常 XPL 计算结果顺序一致；如果历史脏数据顺序不一致，回撤明细可能错配。摘要区已经按年份匹配。
3. `format_export_file_data()` 对夏普、卡玛和所提诺使用 `:.2`，这是两位有效数字格式。多产品 `_fmt_value()` 对数字类指标使用两位小数并去掉尾零。两者可能存在显示精度差异，但底层数值来源一致。
4. 多产品“平均月超额”会忽略缺少 `monthly_excess_return_diff` 的异常月记录；XPL 严格导出遇到这类脏数据可能直接失败。正常 XPL 生成的数据不会有该差异。
5. 多产品组合计算依赖 `TaskResultReturn.returns_json`。清理历史数据时不能只保留 `TaskResult` 而删除收益曲线关联。
6. `format_export_file_data()` 中回撤类摘要使用字符串拼接负号，例如 `f"-{max_drawdown:.2%}"`。在正常 XPL 生成结果中，`start_drawdown` 和 `excess_drawdown_winning_rate` 是非负数，拼负号只是展示约定；但 `max_drawdown = start_drawdown - index_drawdown` 可能为负，字符串拼负号会形成双负号。单产品预览会对展示值做归一化兜底，多产品预览已使用数值取反保持口径一致。后续如允许调整 XPL 导出函数，应优先改为 `f"{-value:.2%}"` 这类数值取反写法。

## 10. 推荐排障顺序

遇到横杠时按以下顺序检查：

1. 确认横杠是否属于第 2.1 节中的设计空值。
2. 确认任务类型：单产品还是多产品。
3. 确认当前 `TaskResult.success`。
4. 检查 `TaskResult.result[*].calculate_metrics` 是否存在。
5. 按第 3 节定位具体字段。
6. 如果是多产品比例列，检查 `TaskResult.return_series_id` 和 `TaskResultReturn.returns_json`。
7. 如果是多产品组合列，检查所有产品是否都有收益序列以及共同日期。
8. 如果只缺少比率类指标，检查是否出现零波动率、零回撤或样本不足。
9. 如果单产品多个指标同时为空，检查是否进入 fallback，以及 `start_end_date` 格式是否正确。
10. 如果数据结构完整但页面仍为空，再检查 API 返回值和模板渲染。
