# 📋 日志系统优化 - 快速使用指南

## 🎯 优化目标

解决数据库膨胀问题（300MB → 5MB），提升系统性能。

## ⚡ 立即开始

### 第一步：备份数据库（重要！）

```bash
# Windows
copy instance\app.db instance\app.db.backup

# Linux
cp instance/app.db instance/app.db.backup
```

### 第二步：运行清理脚本

```bash
python cleanup_task_logs.py
```

按提示操作即可。

### 第三步：验证效果

清理后，你会看到：
- ✅ 数据库从 ~300MB 缩小到 ~5MB
- ✅ 系统运行更流畅
- ✅ 日志功能完全正常

## 🔍 验证是否成功

### 1. 检查数据库大小

```bash
# Windows PowerShell
dir instance\app.db | select Name, Length

# Linux
ls -lh instance/app.db
```

应该看到数据库大小大幅减小。

### 2. 测试日志功能

1. 访问管理面板：http://localhost:5000/admin/logs
2. 创建一个新任务
3. 查看任务日志 - 应该正常显示

### 3. 查看日志文件

```bash
# 查看最新日志
# Windows
type logs\app.log | more

# Linux
tail -f logs/app.log
```

## ❓ 常见问题

### Q: 会丢失数据吗？
**A: 不会**。日志一直在文件中，数据库里的日志是冗余的。

### Q: 需要修改代码吗？
**A: 不需要**。所有改动已完成，API 接口保持不变。

### Q: 如果出问题怎么办？
**A: 恢复备份即可**：
```bash
# Windows
copy instance\app.db.backup instance\app.db

# Linux
cp instance/app.db.backup instance/app.db
```
