# C5服务增强功能实现方案

## 功能概述

本次增强为C5服务添加了以下功能：
1. 统一异常类系统
2. 失败重排与任务级重试机制
3. 可观测性与详细日志
4. 执行统计报表

## 1. 统一异常类系统

### 位置
- `app/exceptions/c5_exceptions.py`
- 已集成到 `app/exceptions/__init__.py`

### 异常类层次结构
```
C5BaseException (基础异常类)
├── C5NetworkException (网络异常)
│   └── C5RateLimitException (429速率限制)
├── C5ExecutionException (执行异常)
│   └── C5ValidationException (验证异常)
├── C5TimeoutException (超时异常)
└── C5DataException (数据异常)
```

### 使用方式
```python
from app.exceptions import C5NetworkException, C5RateLimitException

try:
    # 执行操作
    pass
except ConnectionError as e:
    raise C5NetworkException(
        message="网络连接失败",
        http_status=500,
        request_id="req_123",
        details={"original_error": str(e)}
    )
except Exception as e:
    if "429" in str(e):
        raise C5RateLimitException(
            message="触发速率限制",
            retry_after=60,
            request_id="req_123"
        )
```

## 3. 失败重排与任务级重试机制

### 机制说明
1. **首次执行**：所有参数组合按顺序执行
2. **失败收集**：失败的组合记录到重试队列
3. **重试轮次**：最多M轮（默认M=2）重试
4. **混洗策略**：每轮重试前混洗队列，避免热点
5. **延迟策略**：每轮重试之间整体延迟（默认30秒）
6. **幂等性**：通过step_index和retry_round保证幂等

### 实现逻辑
```python
def get_bdl_with_retry(self, task, name, parameters, config_data):
    max_retry_rounds = config_data.get('max_retry_rounds', 2)
    retry_delay = config_data.get('retry_delay', 30)
    
    # 1. 首次执行所有组合
    all_combinations = self._get_all_combinations(...)
    failed_queue = []
    success_count = 0
    
    for combination in all_combinations:
        try:
            success, result = self._execute_combination(combination)
            if success:
                success_count += 1
            else:
                failed_queue.append(combination)
        except Exception as e:
            failed_queue.append((combination, e))
    
    # 2. 重试轮次
    for round_num in range(1, max_retry_rounds + 1):
        if not failed_queue:
            break
        
        # 混洗队列
        random.shuffle(failed_queue)
        
        # 延迟
        time.sleep(retry_delay)
        
        # 执行重试
        new_failed_queue = []
        for item in failed_queue:
            combination = item[0] if isinstance(item, tuple) else item
            try:
                success, result = self._execute_combination(
                    combination, 
                    retry_round=round_num
                )
                if success:
                    success_count += 1
                else:
                    new_failed_queue.append(item)
            except Exception as e:
                new_failed_queue.append((combination, e))
        
        failed_queue = new_failed_queue
    
    return success_count, len(failed_queue)
```

## 5. 执行统计报表

### 统计指标
- 总请求数
- 成功数
- 失败数
- 429次数（速率限制）
- 平均耗时
- 95分位耗时
- 99分位耗时
- 各异常类型统计

### 实现方式
```python
def generate_statistics_report(self, task_id: str) -> Dict:
    results = TaskResult.query.filter_by(task_id=task_id).all()
    
    execution_times = [r.execution_time for r in results if r.execution_time]
    error_types = [r.error_type for r in results if r.error_type]
    http_statuses = [r.http_status for r in results if r.http_status]
    
    return {
        "total_requests": len(results),
        "success_count": sum(1 for r in results if r.success),
        "failed_count": sum(1 for r in results if not r.success),
        "rate_limit_count": sum(1 for s in http_statuses if s == 429),
        "avg_execution_time": statistics.mean(execution_times) if execution_times else 0,
        "p95_execution_time": numpy.percentile(execution_times, 95) if execution_times else 0,
        "p99_execution_time": numpy.percentile(execution_times, 99) if execution_times else 0,
        "error_type_distribution": Counter(error_types),
        "http_status_distribution": Counter(http_statuses)
    }
```
