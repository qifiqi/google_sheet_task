## 1. 项目概述

本项目是一个基于 **Flask + Vue 3** 的任务执行平台，核心能力包括：

- Google Sheet 参数校验任务（C3 / C4 / C5 多模板）
- C31 批量拆分为多个 C3 子任务
- 回测训练任务与多产品回测
- 任务调度、看门狗自动恢复、断点重启
- 管理后台、模板系统、任务日志和结果查询

### 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Python 3.9 + Flask |
| 数据库 | PostgreSQL 16（开发/生产），SQLite（测试） |
| ORM | Flask-SQLAlchemy + Flask-Migrate |
| 前端框架 | Vue 3 + Vite |
| UI 组件库 | Element Plus + Naive UI |
| 任务执行 | 后台线程 + gspread |
| 生产部署 | Gunicorn + Docker |

## 3. 本地访问地址与端口配置

### 3.1 端口总览

| 服务 | 端口 | 访问地址 | 说明 |
|------|------|---------|------|
| Flask 后端 | `5000` | `http://localhost:5000` | 后端 API + 传统模板页面 |
| Vite 前端开发服务器 | `3000` | `http://localhost:3000` | Vue 3 前端（开发模式） |
| PostgreSQL | `5432` | `127.0.0.1:5432` | 数据库 |
| Gunicorn（生产） | `5000` | `http://0.0.0.0:5000` | 生产模式部署 |
| Nginx 反向代理（生产） | `80` | `http://localhost` | 生产环境前端 + API 反代 |

### 3.2 本地开发模式访问

开发环境下，前后端分别运行：

- **前端页面**：浏览器访问 `http://localhost:3000`
- **后端 API**：Vite 开发服务器会自动将 API 请求代理到 `http://127.0.0.1:5000`
- **传统模板页面**（Jinja2）：直接访问 `http://localhost:5000`

### 3.3 前端 Vite 代理规则

Vite 开发服务器内置了以下反向代理规则，定义在 `frontend/vite.config.js` 中：

| 前端请求路径 | 代理目标 | 说明 |
|-------------|---------|------|
| `/api/*` | `http://127.0.0.1:5000` | 通用 API 接口 |
| `/admin/api/*` | `http://127.0.0.1:5000` | 管理后台 API |
| `/backtest-training/api/*` | `http://127.0.0.1:5000` | 回测训练 API |
| `/xpl/analyze` | `http://127.0.0.1:5000` | XPL 分析 |
| `/xpl/export` | `http://127.0.0.1:5000` | XPL 导出 |
| `/xpl/v1/analyze` | `http://127.0.0.1:5000` | XPL v1 分析 |

如需新增代理路径，编辑 `frontend/vite.config.js` 的 `server.proxy` 段：

```javascript
// frontend/vite.config.js
export default defineConfig({
  server: {
    port: 3000,
    proxy: {
      '/your-new-api-path': {
        target: 'http://127.0.0.1:5000',
        changeOrigin: true,
      },
    },
  },
})
```

### 3.4 生产环境 Nginx 反向代理

生产部署时通过 Nginx 统一入口：

```nginx
server {
    listen 80;
    server_name localhost;

    # Vue 前端静态文件
    root /path/to/谷歌参数批量校验/frontend/dist;
    index index.html;

    # API 请求反代到 Flask
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
    }

    # Vue Router history 模式
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 7. 代理框架配置

本项目涉及两类代理：**Google Sheet API 代理**和**东方财富 (DFCF) K线接口代理**。

### 7.1 Google Sheet API 代理

#### 工作原理

`GoogleSheet` 客户端（`app/services/google_sheet_client.py`）在初始化时，如果传入 `proxy_url`，会将代理注入到 gspread 底层 HTTP Session 中。

#### 代理来源

代理地址存储在数据库 `SystemConfig` 表中，配置 key 为 `proxy_url`：

| 配置 Key | 说明 | 默认值 |
|---------|------|--------|
| `proxy_url` | Google Sheet 请求代理地址 | `None`（直连） |

#### 设置方法

**方法一：通过管理后台**

访问 `http://localhost:5000/admin/config`（或前端 `http://localhost:3000/admin/config`），找到 `proxy_url` 配置项，填入代理地址。

**方法二：直接修改数据库**

```sql
UPDATE system_configs SET value = 'http://user:pass@proxy-host:port' WHERE key = 'proxy_url';
```

#### 代理格式

支持以下格式：

```
http://user:password@host:port
https://user:password@host:port
socks5://user:password@host:port
```

#### 代理生效流程

```
任务创建 → 从 SystemConfig 读取 proxy_url
         → 传给 GoogleSheet.__init__(proxy_url=...)
         → _apply_proxy_settings() 注入到 gspread session
         → 后续所有 Google Sheet API 请求走代理
```

#### 注意事项

- 代理通过 `os.environ['HTTP_PROXY']` 和 `session.proxies` 双重注入
- 任务结束后会调用 `_clear_proxy_settings()` 清理环境变量
- 代理连接失败时，客户端会尝试重连并抛出 `RetryableNetworkTaskError`

### 7.2 东方财富 (DFCF) K线接口代理

#### 工作原理

DFCF API 客户端（`app/utils/dfcf_api.py`）使用 `SmartProxyManager`（`app/utils/proxy_manager.py`）管理短生命周期代理池。

#### 代理开关

| 配置 Key | 说明 | 默认值 |
|---------|------|--------|
| `dfcf_kline_proxy_enabled` | 是否启用 DFCF K 线代理 | `False`（直连） |

#### 开启方法

通过管理后台将 `dfcf_kline_proxy_enabled` 设为 `true`。

#### 代理池机制

`SmartProxyManager` 的核心策略：

```
每次请求 → get_best_proxy()
    ├── 检查代理缓存是否过期（30秒有效期）
    ├── 检查使用次数是否达到上限（50次）
    ├── 命中缓存 → 复用当前代理
    └── 未命中 → 从代理池 API 拉取新代理
```

#### 代理池配置

代理池配置硬编码在 `SmartProxyManager.__init__` 中：

```python
self.proxy_config = {
    "base_username": "<用户名>",
    "password": "<密码>",
    "url": "https://share.proxy.qg.net/get?key=...&num=1&format=json",
}
```

#### 代理失败处理

```
请求失败 → _refresh_proxy_after_failure()
    ├── 重建 HTTP Session
    ├── 使当前代理失效 (invalidate_proxy)
    └── 强制刷新获取新代理 (force_refresh=True)
```

### 7.3 代理配置总结

| 代理类型 | 适用场景 | 配置位置 | 开关 |
|---------|---------|---------|------|
| Google Sheet 代理 | 访问 Google Sheet API | `SystemConfig.proxy_url` | 填值即启用 |
| DFCF K线代理 | 东方财富 K 线数据拉取 | `SystemConfig.dfcf_kline_proxy_enabled` | `true`/`false` |

### 10.2 函数复杂度控制

**规则：单个函数循环复杂度不超过 15，行数不超过 60。**

```python
# ❌ 反模式：复杂度 47 的巨型函数
def _build_c3_summary_rows(results, config):
    # 221 行代码，包含多层嵌套 for/if/try
    for result in results:
        for metric in metrics:
            if condition1:
                if condition2:
                    ...

# ✅ 推荐：拆分为多个小函数
def _build_c3_summary_rows(results, config):
    """构建 C3 汇总行数据。"""
    rows = []
    for result in results:
        row = _build_single_c3_row(result, config)
        if row:
            rows.append(row)
    return rows

def _build_single_c3_row(result, config):
    """构建单条 C3 汇总行。"""
    metrics = _parse_result_metrics(result)
    return _format_c3_row(metrics, config)
```

### 10.3 注释规范

**规则：所有 public 函数和类必须包含 docstring，注释率目标 ≥3%。**

```python
# ❌ 无注释
def process_task(task_id, config):
    ...

# ✅ 标准 docstring

    """处理参数校验任务。

    Raises:
        RetryableNetworkTaskError: 网络异常，可由看门狗自动重试。

    """
    ...
```

### 10.4 错误处理规范

**规则：不吞掉异常，使用 `raise` 保留异常链，任务错误只写原始摘要。**

```python
# ❌ 反模式
try:
    result = risky_operation()
except Exception as e:
    pass  # 吞掉异常
    raise e  # 丢失异常链

# ✅ 推荐
try:
    result = risky_operation()
except RetryableNetworkTaskError:
    raise  # 保留完整异常链
except Exception as exc:
    # 看门狗按 status=error 统一重启，不在业务任务里构造可重启前缀
    root = unwrap_exception(exc) or exc
    error_msg = f"{root.__class__.__name__}: {root}"
    logger.error(error_msg)
    task.error_message = error_msg
    raise
```

### 10.5 代码去重规范

### 10.7 路由层规范

**规则：路由函数（Blueprint handler）只做请求解析、参数校验和响应封装，业务逻辑必须下沉到 Service 层。**

```python
# ❌ 路由层包含大量业务逻辑
@bp.route('/results/<task_id>')
def get_results(task_id):
    # 100 行数据处理代码...
    return jsonify(data)

# ✅ 路由层只做薄壳
@bp.route('/results/<task_id>')
@login_required
def get_results(task_id):
    """获取任务结果详情。"""
    service = TaskResultService()
    result = service.get_result_detail(task_id)
    if result is None:
        return jsonify({"status": "error", "message": "任务不存在"}), 404
    return jsonify({"status": "success", "data": result})
```

## 11. 常见问题排查

### 11.1 启动失败

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| `ModuleNotFoundError` | 未安装依赖 | `pip install -r requirements.txt` |
| `connection refused` 数据库 | PostgreSQL 未启动 | `cd dockers/postgres && docker compose up -d` |
| 端口被占用 | 5000 端口冲突 | 修改 `.env` 中 `PORT` 或终止占用进程 |
| 中文乱码 | PowerShell 编码问题 | 执行本文 §2.2 的编码配置命令 |

### 11.2 前端代理不生效

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| API 返回 404 | 后端未启动 | 确认 `python run.py` 正在运行 |
| CORS 错误 | 直连后端未走代理 | 通过 `http://localhost:3000` 访问 |
| 新增路径 404 | 代理规则未配置 | 在 `vite.config.js` 添加代理规则 |

### 11.3 Google Sheet 连接失败

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| `Credentials` 错误 | Token 文件缺失或过期 | 检查 `data/token.json` 是否存在 |
| 超时 | 网络不通或无代理 | 配置 `proxy_url` 或检查网络 |
| `[NETWORK_RETRYABLE]` | 可恢复网络异常 | 看门狗会自动重试，也可手动重启 |

### 11.4 测试运行

```powershell
# 全量测试
pytest

# 指定测试文件
pytest tests/test/test_p0_p1_refactor.py

# 指定单个测试函数
pytest tests/test_specific.py::test_name
```

> **注意**：部分测试文件（如 `google_sheet_test.py`、`token_test.py`）依赖本地 token 文件或外部环境，全量测试失败时不要第一时间误判为代码改动引入。
